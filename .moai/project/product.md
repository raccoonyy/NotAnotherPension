## @DOC:MISSION-001 핵심 미션

> **"국민연금 vs 개인연금, 똑같은 돈으로 뭐가 더 유리할까?"**

NotAnotherPension은 **한국의 국민연금과 개인연금(IRP, 연금저축)의 예상 수령액을 세후 기준으로 비교**할 수 있는 정적 웹페이지입니다.

### 핵심 가치 제안

#### 4가지 핵심 가치

1. **동일 조건 비교**: 국민연금에 납입하는 9%를 개인연금에 투자했다면? 가정 비교
2. **세후 수령액 중심**: 세금까지 반영한 실질 수령액 비교
3. **연도별 시각화**: 현재부터 60세까지 가로 스크롤 테이블로 한눈에 확인
4. **빌드 없는 단순함**: HTML 파일 하나로 즉시 실행 가능

## @SPEC:USER-001 주요 사용자층

### 1차 사용자: 근로자 (Laborer)
- **대상**: 직장인, 4대보험 가입자
- **입력 정보**: 현재 나이, 현재 연봉 (세전)
- **핵심 니즈**: 회사가 절반 내주는 국민연금 vs 내가 직접 넣는 개인연금 비교
- **특징**: 본인 부담 4.5% + 회사 부담 4.5% = 총 9% 적립

### 2차 사용자: 사업자 (Business Owner)
- **대상**: 자영업자, 프리랜서, 개인사업자
- **입력 정보**: 현재 나이, 월평균 사업소득, 국민연금 최근 월 납부액
- **핵심 니즈**: 본인이 전액(9%) 부담하는 국민연금의 효율성 검토
- **특징**: 소득과 신고액이 다를 수 있음

## @SPEC:PROBLEM-001 해결하는 핵심 문제

### 우선순위 높음
1. **국민연금 vs 개인연금 동일 조건 비교 불가**: 기존 계산기는 각각 따로 계산
2. **세후 수령액 미반영**: 대부분 세전 금액만 표시, 실질 수령액 파악 어려움
3. **임금 상승 곡선 미반영**: 현재 소득 기준 단순 계산, 연령별 임금 변화 무시

### 우선순위 중간
- 근로자/사업자 구분 없는 단일 계산기
- 개인연금 상품별 수익률/수수료 차이 미반영

## @DOC:STRATEGY-001 차별점 및 강점

### 경쟁 솔루션 대비 강점

1. **동일 금액(9%) 기준 비교**
   - 국민연금에 들어가는 돈과 똑같은 금액을 개인연금에 투자했다면?

2. **임금 상승 곡선 (Wage Curve) 반영**
   - 30대: 연 5% 상승
   - 40대: 연 3% 상승
   - 50대 초반: 동결
   - 55세 이후: 임금피크 하락

3. **세후 수령액 중심 표시**
   - 국민연금: 연금소득 간이세액표 적용
   - 개인연금: 연금소득세 5.5%~3.3% 적용

4. **다양한 개인연금 상품 비교**
   - S&P500 ETF (고수익/고위험)
   - TDF 2050 (중수익/중위험)
   - 예금형 초안전 상품 (저수익/저위험)

## @SPEC:SIMULATION-001 시뮬레이션 로직

### A. 임금 상승 곡선 (Wage Curve)

```javascript
const WAGE_GROWTH_RATES = {
  20: 0.05,  // 20대: 연 5% 상승
  30: 0.05,  // 30대: 연 5% 상승
  40: 0.03,  // 40대: 연 3% 상승
  50: 0.00,  // 50대 초반: 동결
  55: -0.02  // 55세 이후: 임금피크 2% 하락
};
```

### B. 국민연금 예상 수령액 (NPS)

**산식**: 기본연금액 = (A + B) × (1 + 0.05n) / 12 × 가입월수/12

- **A값**: 전년도 전체 가입자 평균소득 (data.json에서 로드)
- **B값**: 가입자 본인의 기준소득월액 평균
- **소득대체율**: 현행 40% → 점진적 하락 반영
- **근로자**: 본인 4.5% + 회사 4.5% = 9%
- **사업자**: 본인 9% 전액 부담

### C. 개인연금 비교군 (Private Pension)

**전제**: 국민연금 납입액(9%)과 동일한 금액을 개인연금에 투자

| 상품 | 예상 수익률 | 수수료(보수율) | 특징 |
|------|------------|---------------|------|
| S&P500 ETF | 연 8% | 0.03% | 고수익/고위험 |
| TDF 2050 | 연 5% | 0.50% | 자동 리밸런싱 |
| 예금형 | 연 2.5% | 0% | 원금 보장 |

**복리 계산**: 연도별 (납입액 × (1 + 수익률 - 수수료)^연수)

### D. 세금 계산 (Taxation)

**납부 단계**: 세액공제 효과 미반영 (복잡성 감소)

**수령 단계**:
- **국민연금**: 연금소득 간이세액표 적용
- **개인연금**: 연금소득세 5.5% (70세 미만) ~ 3.3% (85세 이상)

## @SPEC:SUCCESS-001 성공 지표

### 즉시 측정 가능한 KPI

1. **계산 완료율**: 입력 시작 → 결과 표시 80% 이상
2. **체류 시간**: 결과 페이지 평균 2분 이상 (테이블 스크롤 확인)
3. **재방문율**: 7일 내 재방문 20% 이상

### 측정 주기
- **일간**: 페이지뷰, 계산 완료 수
- **주간**: 재방문율
- **월간**: MAU

## EARS 요구사항 예시

```markdown
### Ubiquitous Requirements (기본 기능)
- 시스템은 국민연금 예상 수령액 계산 기능을 제공해야 한다
- 시스템은 개인연금 예상 수령액 계산 기능을 제공해야 한다
- 시스템은 세후 수령액을 기본으로 표시해야 한다

### Event-driven Requirements (이벤트 기반)
- WHEN 사용자가 직업 구분을 변경하면, 시스템은 입력 폼을 해당 직업에 맞게 전환해야 한다
- WHEN 사용자가 계산하기 버튼을 클릭하면, 시스템은 연도별 결과 테이블을 생성해야 한다

### State-driven Requirements (상태 기반)
- WHILE 근로자 모드일 때, 시스템은 본인+회사 부담금을 구분하여 표시해야 한다
- WHILE 결과 테이블이 표시된 상태일 때, 시스템은 가로 스크롤을 지원해야 한다

### Constraints (제약사항)
- 모든 계산은 클라이언트 측에서 수행되어야 한다 (서버 의존 없음)
- 빌드 과정 없이 index.html 파일 하나로 실행 가능해야 한다
```
